# commit表情拟定规范

| 表情               | 意义   | 场景                                                        | 代码                            |
| ------------------ | ------ | ----------------------------------------------------------- | ------------------------------- |
| :art:              | 待补充 | 1.格式化代码2.更改目录结构                                  | :art<span>:</span>              |
| :lipstick:         | 待补充 | 更改UI界面相关内容                                          | :lipstick<span>:</span>         |
| :pencil:           | 待补充 | 更新了文档内容                                              | :pencil<span>:</span>           |
| :sparkles:         | 待补充 | 完成了新的功能(API,烂机器),有标志性的东西                   | :sparkles<span>:</span>         |
| :rocket:           | 待补充 | 1.仅仅更新了代码(还没有完成某一功能或者API)2.部署接口到页面 | :rocket<span>:</span>           |
| :lock:             | 待补充 | 更新的代码用于修复安全问题或者代码作用于安全相关的问题      | :lock<span>:</span>             |
| :fire:             | 待补充 | 1.删除了代码或者文件                                        | :fire<span>:</span>             |
| :bug:              | 待补充 | 修复某个已知bug的代码                                       | :bug<span>:</span>              |
| :zap:              | 待补充 | 优化了代码提升性能                                          | :zap<span>:</span>              |
| :white_check_mark: | 待补充 | 更新测试代码                                                | :white_check_mark<span>:</span> |
| :wrench:           | 待补充 | 更新工具类代码                                              | :wrench<span>:</span>           |
| :poop:             | 待补充 | 代码能跑,没有功能上的问题,只是代码很丑,后期会进行优化       | :poop<span>:</span>             |
| :truck:            | 待补充 | 移动或者重命名了文件                                        | :truck<span>:</span>            |
| :egg:              | 待补充 | 在代码里加了彩蛋(不影响用户体验,不然就是代码里下毒了)       | :egg<span>:</span>              |
| :see_no_evil:      | 待补充 | 更改了.gitignore文件                                        | :see_no_evil<span>:</span>      |
| :heavy_plus_sign:  | 待补充 | 新增了依赖的第三方库文件(.js或者.jar)                       | :heavy_plus_sign<span>:</span>  |
| :heavy_minus_sign: | 待补充 | 移除了依赖的第三方库文件(.js或者.jar)                       | :heavy_minus_sign<span>:</span> |
| :recycle:          | 待补充 | 重构已知代码                                                | :recycle<span>:</span>          |


## FAQ
* 如果commit的代码,做了上面多个表情涵盖的内容怎么办?
  * 选**最主要的内容**对应的表情,其余改动内容用文字描述在commit文本中
